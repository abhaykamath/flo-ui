# FEEDBACK.md — flo-ui

This file is a living record of real mistakes made during development and the lessons learned from them. Claude must read this at the start of every session. When a new mistake is identified, it gets added here with enough context to prevent it from happening again.

Entries are added in reverse chronological order — newest at the top.

---

## How To Read This File

Each entry has:
- **What happened** — the concrete mistake
- **Why it's wrong** — the principle it violated
- **The rule** — what to do instead, permanently

---

## Mistakes Log

*This file starts empty. Entries will be added here as development progresses and real mistakes are identified. Do not populate this with hypothetical mistakes — only real ones observed during actual sessions.*

---

## How To Add An Entry

When Abhay identifies a mistake or pattern worth recording, add it here in this format:

```
### [Short title of the mistake]
**Date:** YYYY-MM-DD
**What happened:** One or two sentences describing exactly what Claude did wrong.
**Why it's wrong:** The principle or rule it violated.
**The rule:** The concrete, permanent thing to do instead.
```

---

## Notes

- This file grows over time. It is not cleaned up or summarised — every entry stays permanently.
- If an entry here conflicts with CLAUDE.md or RULES.md, the entry here takes precedence — it represents a real-world correction to a rule that wasn't strict enough.
- If the same mistake appears twice, the rule was not clear enough. Rewrite the rule in RULES.md and note it here.
